分离内核架构

一台机器，多个内核，毫不妥协。

Multikernel 系统为每个应用程序提供独立的 Linux 内核，拥有专用核心和原生硬件性能。设备驱动与 I/O 被卸载到独立的内核处理。无需虚拟机管理程序，没有虚拟化开销，也没有共享内核带来的攻击面。

为何选择分离内核架构

隔离能力内建于架构之中

在传统 Linux 内核中，应用程序代码与设备处理会争用相同的 CPU 核心。分离内核架构将全部设备处理卸载到专用核心上的专用内核，从而消除了这种争用。应用程序感受不到任何中断——这并非调优的结果，而是因为中断在物理上根本无法到达它们。
